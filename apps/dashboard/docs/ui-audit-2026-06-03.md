# Fincept Terminal — UI/UX Audit & Upgrade Plan

**Date:** 2026-06-03
**Auditor:** Terminal UI Lead
**Scope:** `apps/dashboard` (Next.js 14 · TypeScript · Tailwind · Radix · TanStack Query)
**Routes audited:** `/`, `/positions`, `/orders`, `/strategies`, `/reconciliation`, `/portfolio-builder`, `/news`, `/research`, `/news-impact-lab`, `/predictions`, `/markets`, `/models`, `/risk`, `/system`, `/signal-cockpit-demo`, `/login`.

---

## Executive Summary

Fincept Terminal already has a **strong design DNA**: dense monospace typography, semantic color tokens (cyan = verified, green = long, red = short, amber = warn, purple = AI), and a Bloomberg-style shell. The previous audit (`aestheticaudit.md`, 2026-05-09) called the cockpit demo "the best contrast surface" — the team is on the right track.

However, the **connected product still reads as "dashboard with cards"**, not "professional trading terminal." Five issues hold it back:

1. **Visual identity mismatch.** The brief calls for *cobalt + orange* on a *glass/gunmetal OLED* base with *dot-matrix/LED* details. The current palette is cyan-dominated and uses flat borders, no glass, no LED grammar.
2. **No watchlist surface.** Bloomberg, TradingView, Fidelity Active Trader — every pro trading product has a watchlist as a first-class screen. We don't have one.
3. **No per-symbol detail page.** `/markets` lists symbols, but clicking a symbol goes nowhere. The most important screen in a trading terminal is the symbol detail view; we don't have it.
4. **Charts are minimal.** A single LineChart with a tooltip. No volume, no range chips, no compare-draw, no crosshair, no last-trade tape.
5. **Mock/live ambiguity.** Mock data looks identical to live data; the parent and the dad-as-operator use case both demand that mock state is unmistakable.

This plan delivers all five in a single coherent slice, **without breaking any existing functionality**.

## Audit by Area

### Visual Hierarchy

- **Strong:** The shell is consistent (TitleBar + SafetyStateBar + NavTabs + StatusBar). Status language is consistent (READY / WATCH / DEGRADED). Page headers are uniform.
- **Weak:** Most pages open with a long PageHeader description, then a 2–4 KPI strip, then a single card. The first-fold rarely tells a story.
- **Weak:** Card density varies wildly. `/portfolio-builder` and `/news-impact-lab` are visually quiet on first load.
- **Weak:** `Card` is a flat-bordered box. There's no glass, no inner glow, no subtle scanline — which the brief explicitly asks for.

### Color & Accent

- **Current:** Cyan (verified), green (long/profit), red (short/loss), amber (warn), purple (AI). Plus the existing UI primary which is bright cyan.
- **Wanted:** Cobalt blue + orange as the new primary/secondary. OLED black base, glass/gunmetal panels.
- **Recommendation:** Keep existing semantic tokens (they are well-designed) but **add a parallel "brand" token set** for the new visual language. The new tokens become the default for hero sections, signal cards, and trading-chart accents; existing semantic tokens stay for status (don't break the status grammar).

### Typography

- **Strong:** Monospace everywhere. Tabular-nums. Uppercase labels. Small dense type. Consistent.
- **Weak:** Body description text uses `text-muted-foreground` which is fine, but the long descriptions in PageHeader often get truncated or wrap awkwardly.
- **Recommendation:** No change. Strengthen hierarchy by adding `font-display` weight (700) for KPI values and `font-mono` numeric tabulation (already in place).

### Charts

- **Current:** Single LineChart on `/markets`. No volume, no range selector (1D/1W/1M/3M/1Y/ALL), no crosshair, no last-trade tape.
- **Recommendation:** Build a `TradingChart` component with:
  - Range chip selector
  - Main area line (cobalt + orange alternatives)
  - Volume bars at the bottom
  - Crosshair on hover
  - Last trade tape on the right
  - Y-axis with current price pin
  - Mock-data disclosure chip in the corner

### Empty / Loading / Error States

- **Strong:** `EmptyState` component exists and is used.
- **Weak:** `EmptyState` is one component for every context — loading, no data, error, "not yet" all look the same.
- **Weak:** Loading states are inconsistent: some pages show nothing while loading, some show shimmering skeletons (none), some show stale data.
- **Recommendation:** Add `LoadingState` and `ErrorState` variants. Add a global `MockBadge` for mock data.

### Mobile / Responsiveness

- **Weak:** Most grids break at small viewports. The TitleBar wraps but produces cramped controls. The NavTabs overflow-x but no hamburger fallback.
- **Recommendation:** No mobile rewrite in this slice. Add minimum responsive guards to new components only.

### Navigation

- **Strong:** Both NavTabs (horizontal) and Sidebar (vertical) exist. AppShell uses NavTabs.
- **Note:** The Sidebar is unused (AppShell uses NavTabs) but the file remains — dead code we can keep for now.
- **Recommendation:** No change.

### Accessibility

- **No findings** — keyboard nav on dialogs works (Radix), kbd shortcuts on command palette, focus rings on buttons. Reasonable baseline.
- **Recommendation:** Ensure new components maintain parity.

## Implementation Strategy (this slice)

1. **Tokens** — Add `brand` color tokens (cobalt + orange), `surface` tokens (gunmetal + glass), `led` glow utilities.
2. **Atoms** — `MockBadge`, `LEDDot`, `DotMatrix`, `SignalCard`, `TradingChart`.
3. **Pages** — `/watchlist` (new), `/symbol/[symbol]` (new), restyle of `/` dashboard hero.
4. **Sanity** — Typecheck, existing tests must still pass.

No existing component gets removed. No existing color semantic is renamed. The brand palette is **additive** — current pages keep their look unless explicitly restyled.

---

## Mock Data Discipline

**Rule:** Any data that comes from a mock/seed/demo source gets:

1. A `<MockBadge />` chip in the page header (or relevant panel).
2. A console.warn in dev with `MOCK:` prefix and a link to the source.
3. A comment in the data file: `// MOCK: not backed by API yet — see FIN-XXXX`.

This protects the dad-as-operator trust model. If a panel says `API LIVE` and shows fake numbers, that's a trust collapse.
