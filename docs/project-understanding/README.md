# Project Understanding

This folder is a current code-grounded orientation pack for Fincept Terminal.
It is designed for developers and future coding agents who need to understand
the repository quickly without relying on stale roadmap claims.

| File | Purpose |
|---|---|
| `00-overview.md` | Executive summary, users, workflows, and maturity assessment. |
| `01-goals-and-scope.md` | Confirmed goals, inferred goals, out-of-scope areas, and open questions. |
| `02-architecture.md` | Runtime architecture, boundaries, storage, integrations, auth, and deployment model. |
| `03-codebase-map.md` | Practical repository map with important directories, entry points, configs, tests, and artifacts. |
| `04-features.md` | Confirmed, partial, planned, missing, and unclear features with evidence. |
| `05-data-flow.md` | How market data, signals, orders, API calls, UI state, and artifacts move through the system. |
| `06-current-status.md` | What appears complete, incomplete, broken, runnable, and unverified today. |
| `07-risks-and-issues.md` | Risk register from this audit, grouped by severity. |
| `08-agent-continuation-guide.md` | Safe operating guide for the next AI coding agent. |
| `09-next-steps.md` | Ordered development plan with validation methods. |
| `10-ml-improvement-report.md` | Deep ML-side improvement plan with proven methods and frontier designs. |

Use `08-agent-continuation-guide.md` first when continuing implementation.
Use `07-risks-and-issues.md` and `09-next-steps.md` when choosing the next
safe work slice.
