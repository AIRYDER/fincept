## Executive Summary

Five bullets, severity-tagged. Everything else in this document is evidence.

1. **[CRITICAL — must fix before any live-trading conversation]** **Kill-switch state divergence.** The API writes `control:kill_switch:state` to Redis (`services/api/src/api/routes/control.py:55,242`); the OMS-side `KillSwitchState` (`services/risk/src/risk/state.py:39-73`) **never reads that key** — it only watches in-memory `AlertEvent`s via the bus consumer. The dashboard's `GET /kill-switch` can report `engaged=True` while `check_intent` in the OMS still allows trades. State is also in-memory only, so the kill-switch is lost on restart. This is the single most important production-safety gap in the system and is the reason the 2026-06-23 *Limited Live Readiness Review* correctly returned a NOT-READY verdict. *(Confirmed by Builder 1 round 2 and Scout 1 deep-dive, converging evidence.)*

2. **[CRITICAL — must fix before next CI run]** **Six CI-breaking config/code drifts** that will silently fail or no-op on the next run:

   - `.github/workflows/build-images.yml` references `infra/docker/*` Dockerfiles that **do not exist** (workflow is dead code).

## 2. Must-Fix (CRITICAL + safety HIGH) — 11 items

These are deployment blockers for anything beyond paper replay. Each item names owner, evidence, and fix.

### M-1. Kill-switch state divergence  — **CRITICAL**

- **Where:** `services/api/src/api/routes/control.py:55,242` (Redis writer) vs `services/risk/src/risk/state.py:39-73` (in-memory-only reader).
- **Why:** Operator engages kill switch on dashboard; OMS happily processes orders because the Redis key is invisible to it. State is also in-memory only, so a service restart drops the kill-switch state.
- **Convergent evidence:** Builder 1 round 2 + Scout 1 addendum 2.
- **Fix:** Make `KillSwitchState` read from Redis on every `check_intent` call (or move the source of truth to a single writer/reader pair). Persist kill-switch state to a durable store (Redis with TTL, or DB). Add a startup rehydrate that fails closed if a kill-switch was engaged at last shutdown.
- **Owner:** Backend (risk + api cross-team).

### M-2. Audit failures silently dropped  — **CRITICAL**

- **Where:** `with contextlib.suppress(Exception):` wrapping audit `append()` calls across `services/oms/src/oms/main.py:86-92`, `services/portfolio/...`, `services/orders/...`, `services/strategy_host/...`, `libs/fincept-tools/...`, `libs/fincept-bus/...` (7+ files per Scout 1).
- **Why:** Audit trail becomes best-effort exactly when the system is under stress. No metric exists to detect an audit outage.
- **Fix:** Replace `contextlib.suppress(Exception)` with explicit error handling that logs to stderr, increments a `audit_failures_total` Prometheus counter, and re-raises for fatal paths. Add an SLO: audit failure rate > 0 should page.
- **Owner:** Backend.

### M-3. `next.config.mjs` defaults API URL to `:8000`  — **CRITICAL**

- **Where:** `apps/dashboard/next.config.mjs:4-6`.
- **Fix:** Read `process.env.NEXT_PUBLIC_API_URL` with `:8010` default; rewrite to match `apps/dashboard/src/lib/api.ts:94`.
- **Owner:** Frontend.

### M-4. `.github/workflows/build-images.yml` references Dockerfiles that do not exist  — **CRITICAL**

- **Where:** Workflow matrix points to `infra/docker/{ingestor,agents,api,orchestrator,risk,oms}.Dockerfile`; only `runpod/quant-foundry-{inference,training}/Dockerfile` exist.
- **Fix:** Either generate the missing `infra/docker/*` files, or remove the workflow, or scope it to the RunPod images. Do not leave a `::notice::` masquerading as a build signal.
- **Owner:** DevOps.

### M-5. `scripts/verification-receipt.ps1:124-125` invokes `npm run test:shadow-news-impact`  — **CRITICAL**

- **Where:** Script `apps/dashboard/scripts/run-shadow-news-impact-tests.cjs` exists but is not in `package.json`.
- **Fix:** Add `"test:shadow-news-impact": "node scripts/run-shadow-news-impact-tests.cjs"` to `apps/dashboard/package.json`.
- **Owner:** Frontend.

### M-6. `system-readiness.ts` env-var catalog lists wrong names  — **HIGH (safety-adjacent)**

- **Where:** `apps/dashboard/src/components/system/system-readiness.ts:152-167`.
- **Fix:** Replace catalog with names that match `libs/fincept-core/src/fincept_core/config.py` + `services/api` (see M-7 for the full canonical list).
- **Owner:** Frontend.

### M-7. `.env` contains production-shaped API keys  — **HIGH (security)**

- **Where:** Repo-root `.env` — OpenAI `sk-proj-…`, Anthropic `sk-ant-…`, Alpaca `PKM0K4ZV…`, Polygon, FRED, Exa, Newsapi, Finnhub, Tiingo, Tinker.
- **Fix:** (1) Rotate every key in the `.env` immediately. (2) Replace `.env` with `.env.example` + comments, or leave `.env` blank. (3) Add `scripts/rotate-env.sh` that walks an operator through rotation. (4) Add a CI lint that fails if `.env` matches any of the production-key shapes.
- **Owner:** Security/ops.

### M-8. Hallucinated LLM model defaults in `portfolio-report/route.ts`  — **HIGH (will 404 in production)**

- **Where:** `apps/dashboard/src/app/api/portfolio-report/route.ts` — references `gpt-5.5` and `claude-opus-4.7`.
- **Fix:** Replace with real model IDs (e.g. `gpt-4o` / `claude-3-5-sonnet-latest`) or surface the AI provider/model choice as a server-side config in `libs/fincept-core/src/fincept_core/config.py`.
- **Owner:** Frontend.

### M-9. Path-traversal regressions from 2026-05-16  — **HIGH (security)**

- **Where:** `services/api/src/api/routes/backtest.py:112-148` (bars_path); `services/api/src/api/training.py:252-259` (_validate_input_path).
- **Fix:** Resolve against an allow-listed root (`DATA_DIR` / `FINCEPT_TRAINING_INPUT_ROOT`) using `Path.resolve(strict=True).is_relative_to(root)`; reject with 400 otherwise. The training docstring already promises this behavior — implementation is the only thing missing.
- **Owner:** Backend.

### M-10. WebSocket `?token=` query-string fallback leaks JWTs to access logs  — **HIGH (security)**

- **Where:** `services/api/src/api/ws.py:55-73`.
- **Fix:** Drop the query-string fallback. Exchange a one-time ticket via `POST /auth/ws-ticket` and present it inside the first WS frame.
- **Owner:** Backend.

### M-11. `MAX_DAILY_LOSS_USD` is declared but never enforced  — **HIGH (safety, P0 from 2026-06-03 audit)**

- **Where:** Config at `libs/fincept-core/src/fincept_core/config.py:78`; zero references in `services/risk/src/risk/{checks,snapshot,state}.py`.
- **Fix:** Add a daily-loss check in `risk/checks.py` that compares realized + unrealized PnL against `MAX_DAILY_LOSS_USD` and engages the kill-switch on breach (which then must also be persisted per M-1).
- **Owner:** Backend.

## 3. Should-Fix (HIGH functional) — 10 items

In rough priority order:

- **S-1. `apps/dashboard/src/app/page.tsx` is 997 lines** with a `void OrderStatusBadge;` dead-import silencer at line 995. Split into `components/overview/{kpi-row,feature-launch-panel,activity-feed,live-predictions}.tsx`.
- **S-2. `scripts/start.ps1` and `scripts/start_feature.ps1` are ~99% duplicated** (also `stop.ps1` / `stop_feature.ps1`). Drift has already begun (`start.ps1:491` references nonexistent `./start.bat`). Extract `scripts/_lib/*.ps1` helpers.
- **S-3. OMS Alpaca submit path catches only `AlpacaError`** — `httpx.HTTPError`, `OSError`, `TimeoutError` are uncaught. Network blip → stuck `PENDING_NEW` → duplicate on re-submit. Catch broader; add backoff; classify 4xx vs 5xx in the poll loop. *(`services/oms/src/oms/alpaca/runtime.py:109-122` and `:145-157`.)*
- **S-4. Configuration split** between `pydantic-settings` and 30+ raw `os.environ.get` calls with two competing naming conventions (`FINCEPT_*` vs bare). Add a single source of truth; pick one naming convention; add a startup `assert_safe_for_runtime()` to every service entrypoint (orchestrator, ingestor, strategy_host, portfolio, OMS, features, jobs currently skip it).
- **S-5. `MARK_TTL_SEC` is documented as a `Settings` field but does not exist there** — access raises `AttributeError` before the `or` fallback. `marks.py:50` reads `get_settings().MARK_TTL_SEC or MARK_TTL_SEC`. Either add the field to `Settings` or remove the misleading access.
- **S-6. `_new_order_shell` synthesises `Order` objects with `strategy_id="alpaca.unknown"` and `side=Side.BUY`** that *can be published unchanged* if the Alpaca response is malformed. Validate the response shape; reject malformed; never publish a placeholder order.
- **S-7. Risk caps accept negative values** (`MAX_NOTIONAL_USD_PER_SYMBOL`, `MAX_GROSS_NOTIONAL_USD`, `MAX_DAILY_LOSS_USD` at `config.py:76-78`). Negative → silently disables the gate. Add `field_validator(allow_reuse=True)` to require `>= 0` (or `> 0` for caps).
- **S-8. `MARK_TTL_SEC` default-to-0 in `HttpRunPodClient`** (`runpod_client.py:411` — round-1 audit cited line 273, Reviewer caught the drift). `getattr(client, "cost_per_dispatch_cents", 0)` defaults to 0 even when real cost is unknown — a budget accounting gap. Raise or default to a conservative sentinel.
- **S-9. WebSocket auth-fallback message deserialization skip** — silent failures in event decoder. Log + drop to DLQ.
- **S-10. `preflight.ps1` and `task-check.ps1` do not pin lockfiles** (`uv sync --all-packages --all-groups`, `pnpm install --frozen-lockfile=false`). Local preflight will silently update `uv.lock` and `pnpm-lock.yaml`. Switch to `--frozen`.
