# Fincept Terminal UI Audit - 2026-05-16

Scope: dashboard UI color, layout, and overall visual quality. I inspected the authored code in `apps/dashboard`, then verified the render on desktop and mobile widths in a live browser.

## Overall Verdict

The UI has a strong identity. It feels like a deliberate operator console rather than a generic SaaS dashboard. The terminal language is coherent, the semantic color system is mostly disciplined, and the home screen is already good at telling an operator what matters first.

The main weakness is density. On desktop the design is intentionally compact, but on mobile the fixed shell chrome becomes cramped and starts clipping. A few surfaces also repeat the same dark-card treatment so often that the hierarchy flattens out.

## Findings And Recommendations

### 1. The color system is coherent, but the shell leans too hard on cyan and orange

Observed in:

- [`apps/dashboard/src/app/globals.css`](C:/Users/nolan/CascadeProjects/fincept-terminal/apps/dashboard/src/app/globals.css:13-41)
- [`apps/dashboard/src/components/ui/card.tsx`](C:/Users/nolan/CascadeProjects/fincept-terminal/apps/dashboard/src/components/ui/card.tsx:20-69)
- [`apps/dashboard/src/components/shell/safety-state-bar.tsx`](C:/Users/nolan/CascadeProjects/fincept-terminal/apps/dashboard/src/components/shell/safety-state-bar.tsx:132-180)

Recommendation:

Keep the current semantic palette, but reserve orange for primary/attention states and cyan for verified/system truth. Reduce how often card headers and shell chrome default to cyan.

Pros:

- Stronger state separation.
- Less visual noise in the top chrome.
- Important warnings and success states will read faster.

Cons:

- Slightly less “terminal glow” and less brand personality.
- Some existing surfaces will need a small restyle pass to stay consistent.

### 2. Desktop density is good, but the spacing rhythm is too tight for long reading sessions

Observed in:

- [`apps/dashboard/src/components/ui/card.tsx`](C:/Users/nolan/CascadeProjects/fincept-terminal/apps/dashboard/src/components/ui/card.tsx:24-70)
- [`apps/dashboard/src/components/widgets/page-header.tsx`](C:/Users/nolan/CascadeProjects/fincept-terminal/apps/dashboard/src/components/widgets/page-header.tsx:12-20)
- [`apps/dashboard/src/features/portfolio-builder/PortfolioReportView.tsx`](C:/Users/nolan/CascadeProjects/fincept-terminal/apps/dashboard/src/features/portfolio-builder/PortfolioReportView.tsx:96-240)

Recommendation:

Increase vertical breathing room on top-level content blocks, especially where paragraphs and tables sit together. Keep compact spacing for tables and chrome, but give long-form report sections a bit more line-height and padding.

Pros:

- Easier to read.
- Better hierarchy between sections.
- Less fatigue on dense pages like the optimizer report.

Cons:

- Slightly more scrolling.
- Less information fits above the fold.

### 3. The authenticated shell is strong on desktop, but it breaks down on mobile

Observed in:

- [`apps/dashboard/src/components/shell/app-shell.tsx`](C:/Users/nolan/CascadeProjects/fincept-terminal/apps/dashboard/src/components/shell/app-shell.tsx:16-63)
- [`apps/dashboard/src/components/shell/nav-tabs.tsx`](C:/Users/nolan/CascadeProjects/fincept-terminal/apps/dashboard/src/components/shell/nav-tabs.tsx:43-99)
- [`apps/dashboard/src/components/shell/safety-state-bar.tsx`](C:/Users/nolan/CascadeProjects/fincept-terminal/apps/dashboard/src/components/shell/safety-state-bar.tsx:131-180)
- [`apps/dashboard/src/components/shell/status-bar.tsx`](C:/Users/nolan/CascadeProjects/fincept-terminal/apps/dashboard/src/components/shell/status-bar.tsx:55-101)

Browser evidence:

- Desktop at 1280px looked coherent and intentional.
- Mobile at 390px showed clipped top-bar labels, cramped status chips, and a footer that was mostly reduced to fragments.

Recommendation:

Collapse or hide secondary chrome on narrow widths. Keep the essential state, but move low-priority chips into an overflow menu or a compact summary strip.

Pros:

- Much better mobile legibility.
- Prevents label truncation and visual crowding.
- Keeps the shell usable on small screens.

Cons:

- Some always-visible information would move one click deeper.
- Power users lose a bit of at-a-glance telemetry on phones.

### 4. Several status strips use long copy that gets truncated in exactly the wrong places

Observed in:

- [`apps/dashboard/src/components/shell/safety-state-bar.tsx`](C:/Users/nolan/CascadeProjects/fincept-terminal/apps/dashboard/src/components/shell/safety-state-bar.tsx:132-180)
- [`apps/dashboard/src/components/shell/status-bar.tsx`](C:/Users/nolan/CascadeProjects/fincept-terminal/apps/dashboard/src/components/shell/status-bar.tsx:55-101)

Recommendation:

Shorten the always-visible messages and move the full explanation into tooltips or drill-down panels. The guardrail sentence in the safety bar and the status bar telemetry are both too long for the strip they live in.

Pros:

- Cleaner chrome.
- Fewer clipped labels.
- Better fit for both desktop and mobile.

Cons:

- More reliance on hover/click for detail.
- Slightly less explicit context at a glance.

### 5. The product has two visual modes: a soft login hero and a hard terminal shell

Observed in:

- [`apps/dashboard/src/app/login/page.tsx`](C:/Users/nolan/CascadeProjects/fincept-terminal/apps/dashboard/src/app/login/page.tsx:52-140)
- [`apps/dashboard/src/components/shell/app-shell.tsx`](C:/Users/nolan/CascadeProjects/fincept-terminal/apps/dashboard/src/components/shell/app-shell.tsx:44-63)

Recommendation:

Either intentionally embrace the split or soften the login page so it feels like part of the same system. Right now the login screen is more editorial and atmospheric than the rest of the app.

Pros:

- Better brand continuity.
- The transition into the authenticated shell feels more deliberate.

Cons:

- Less dramatic sign-in page.
- You lose a little of the “welcoming front door” effect.

### 6. The portfolio builder is one of the strongest pages, but it is also the most crowded

Observed in:

- [`apps/dashboard/src/features/portfolio-builder/PortfolioBuilderPage.tsx`](C:/Users/nolan/CascadeProjects/fincept-terminal/apps/dashboard/src/features/portfolio-builder/PortfolioBuilderPage.tsx:94-160)
- [`apps/dashboard/src/features/portfolio-builder/PortfolioReportView.tsx`](C:/Users/nolan/CascadeProjects/fincept-terminal/apps/dashboard/src/features/portfolio-builder/PortfolioReportView.tsx:38-240)

Recommendation:

Keep the current two-pane structure, but vary the section treatment more aggressively. Right now the report side contains many bordered blocks in a row, which is accurate but visually flat. Use a few larger bands or stronger sectional gaps to give the eye a rest.

Pros:

- Better hierarchy inside very dense report pages.
- Makes the “most important” sections pop more clearly.
- Reduces the feeling of a wall of boxes.

Cons:

- More design decisions to maintain.
- Slightly less uniformity across the app.

## Best-Sized Wins To Make First

1. Fix mobile shell clipping.
2. Shorten the long strip messages.
3. Give top-level report sections a little more breathing room.
4. Reduce repeated cyan in card headers and shell chrome.
5. Decide whether the login page should stay editorial or match the terminal shell more closely.

## Bottom Line

This is already a real product UI, not a throwaway dashboard skin. The next jump in quality comes from making the shell more responsive, reducing repeated chrome noise, and giving heavy pages a slightly better reading rhythm. The identity is there. The refinement work is mostly about making that identity hold up outside a wide desktop monitor.
