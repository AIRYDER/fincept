## Why it mattered

The `nightly` workflow on `main` has failed **5+ consecutive scheduled runs**
(2026-07-04 through 2026-07-09). Two jobs fail every night:

1. **Trivy filesystem scan** — 5 HIGH Next.js CVEs + 1 CRITICAL false-positive secret
2. **Python dependency audit (pip-audit)** — "Found 53 known vulnerabilities in 15 packages"

The pip-audit failure was **misdiagnosed** in prior triage receipts
(`receipt-20260704T092500Z.md`, F4): the prior analysis assumed the 53
vulnerabilities were from the project's transitive dependencies and
recommended `uv lock --upgrade`. **This was wrong.**

Root cause: pip-audit's `pip-api` defaults to `/usr/bin/python3` — the
GitHub Actions Ubuntu runner's **system** interpreter — not the project
venv created by `uv sync`. The CI log explicitly warned:

> pip-audit will run pip against /usr/bin/python3, but you have a virtual
> environment loaded at .venv. This may result in unintuitive audits,
> since your local environment will not be audited.

The 15 "vulnerable packages" were Ubuntu OS packages: `cloud-init`, `ufw`,
`bcc`, `sos`, `walinuxagent`, `python-apt`, `distro-info`, etc. — none are
part of the project. The only project-relevant finding (`cryptography
41.0.7`) was also from the system Python; the project's `uv.lock` already
pins `cryptography 47.0.0` (safe).

**Impact of the fix:** eliminates ~50 false positives per nightly run,
restores meaningful security signal, and unblocks the nightly pipeline
greening (pending the Trivy issues, which are separate).
